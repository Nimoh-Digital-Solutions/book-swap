Exchange State Machine

Complete lifecycle of an exchange request — 7 primary states with all transitions mapped to PRD Section 9

BROWSE
User finds a book they want
↓
PENDING
Partner request sent. Owner notified. Can be cancelled by requester.
↓
Owner Response?
Decline
DECLINED
Terminal state. Both books stay Available.
14 Days Timeout
EXPIRED
Terminal state. Auto-triggered by Celery Beat.
User Cancels
CANCELLED
Either user, any time before swap.
Accept
ACCEPTED
Both users prompted to review conditions
↓
CONDITIONS_PENDING
Waiting for both users to agree. Chat locked.
↓
ACTIVE
Both accepted conditions. Chat unlocked. Books marked "In Exchange".
↓
SWAP_CONFIRMED
Both users confirmed physical swap. Rating prompt shown. Swap count ++.
↓
COMPLETED
Archived. Chat read-only. Books remain "In Exchange" until return (P1).

State Transition Reference

From State To State Trigger Who Side Effects
PENDING ACCEPTED Owner clicks Accept Owner Conditions screen shown to both. Other pending requests auto-declined.
PENDING DECLINED Owner clicks Decline Owner Requester notified. Optional decline reason.
PENDING CANCELLED Requester cancels Requester Owner notified. Books stay Available.
PENDING EXPIRED 14 days no response System Both notified. Celery Beat cron job.
ACCEPTED CONDITIONS_PENDING Automatic after accept System Conditions screen presented to both users.
CONDITIONS_PENDING ACTIVE Both users accept conditions Both Chat unlocked. Books → "In Exchange". DB trigger on both rows present.
CONDITIONS_PENDING CANCELLED Either cancels or 14-day timeout Either / System Both notified. Books back to Available.
ACTIVE SWAP_CONFIRMED Both users confirm swap Both Rating prompt. Swap count ++. Chat → read-only.
ACTIVE CANCELLED Either user cancels Either Books back to Available. Cancellation tracked.
SWAP_CONFIRMED COMPLETED Both rated (or 30-day timeout) Both / System Exchange archived. Final state.

Technical Implementation (from Architecture Doc)

Backend state_machine.py with TRANSITIONS dict DB exchange_requests.status enum Trigger DB triggers for conditions + swap confirmation Cron Celery Beat for auto-expire + auto-confirm API ExchangeViewSet with transition actions
